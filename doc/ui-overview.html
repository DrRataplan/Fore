User Interface Overview

The User Interface (UI) is made of Web Components. The UI is enclosed by a single xf-form element. There are two kinds of components - simple controls and container controls. Controls are bound to data items via their bind attribute.

Fore comes with a consistent, ready-made set of the most important components. However it is assumed that the collection grows driven by need.

Native Controls

Fore does not allow to directly use native controls like input or select. Instead these are wrapped into components with a xf- prefix (xf-input, xf-select,...).

The reason for that is simple: it allows to attach the necessary logic for the consistent handling of state and makes sure that the control plays by the rules. This could have been applied to the native controls either but to the price of added complexity in the codebase.

Abstract Control

In Fore the xf-abstract-control implements the core of the data-binding and property updating already. All controls inherit from XfAbstractControl class and usually require minimal code to eventually attach custom eventhandlers or apply custom behavior for updating. New Components can thereby be added with very little effort.

Container Components

Container components may contain other container or simple components.

Component Description
xf-dialog shows a modal dialogbox
xf-group groups a set of controls. As groups can be bound they can change their relevance to hide/show parts of the UI.
xf-repeat Repeat binds to a set of values generating user interface controls for each of them.
xf-switch Defines parts of the user interface to be switchable on and off displaying on of them at a given time.
xf-case Container for a single view defined by a xf-switch.
