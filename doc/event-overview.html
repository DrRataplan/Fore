Events

Events are used to trigger Actions. Events are dispatched either by the form processor (xf-form) or directly by the form author with xf-dispatch.

Events Overview

Event Context Info Description
actions-performed none fired after a block of actions has been executed. When using a server-side submission and the data have been mutated an update request will be send to the server. Otherwise (using client-side submission) no update request is send.
form-ready none dispatched when form is completely initialized
message dispatched by xf-message action
level the message level
message the message to display
model-ready none fired after UI model has been loaded
no-template-error dispatched by repeat when there's no template found
message text message with id of repeat
refresh none fired when UI is about to be refreshed
refresh-done none fired when UI refresh has been completed
repeat-item-appended fired when a new entry has been appended to a repeat
bind the bind id of the repeat
appendLocation the index of the newly appended item
appendedItem the bind object of the appended item
path the binding path of the newly appended item
repeat-item-inserted fired in response to an item being inserted into a repeat
bind the bind id of the repeat
insertLocation the index of the inserted item
insertedItem the binding object of the inserted item
path the binding path of the newly appended item
repeat-item-deleted fired in response to an item being deleted from a repeat
bind the bind id of the repeat
deleteLocation the index of the deleted item
deleteItems the delete binding object
path the binding path of the newly appended item
value-change dispatched in response to a control changing its value
modelItem the binding object being changed
path the binding path of the newly appended item
target the event target element
