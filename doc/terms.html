Terms

This document explains some of the terms used in Fore.

Binding Attributes

Binding AttribuTes are the ref and set attributes of xf-bind as well as the bind attribute on UI Controls. They establish a data-binding between a source (the element having the binding attribute) and a target (where the bind points to).

Bind Objects

At init-time of a form all xf-bind elements are 'unrolled' to create corresponding bind objects. A bind object can be seen as an instanciation of a bind for a single bound node in the data. For single-node bindings each xf-bind will correspond to one bind object. For bindings that bind to a set of items for each of the items a bind object is created. Bind objects are serialized as a JSON structure that represents the UI Model.

Binding Path

Bindings Pathes are generated by the client when the user interacts with the form. They consist of the id of the binding along with an index if the binding refers to repeated data. The concept is explained here.

Host Model

Refers to xf-model element and its descendants. This is the actual model being processed on the server-side by eXist-db to recalculate and revalidate the data. The host model is not transferred to the client but translated into a JSON structure that will make up the UI model

UI Model

The UI Model is powered by a JSON structure that reflects the structure and state of the bindings on the server-side. The UI Model is all the client knows about. The client will mutate this structure when values are changed, items get inserted or deleted. Changes will be logged and send back to the server.
