The Model

The model describes the data, their loading and storing and the constraints to be enforced during validation.

Client and Server Models

There's a model both on the client and the server. However they are not the same and not even carry the same amount of information. To establish a clear distinction between the two i'll call the server-side model the host model and the client-side model the UI model.

It is the host model that a form author will write to define the involved data, constraints and submissions. The host model uses elements with a 'xf-' prefix. The UI model will be created by the backend at runtime.

The host model will never be exposed to the client. When a new form session starts the xf-model element is filtered out before returning the embedding HTML page.

xf-model Overview

Element Description Cardinality
xf-model Container element defining the data, constraints and submissions of a form exactly one
In XForms a form can hold any number of model elements. However in practice more than one model have barely been needed during years of XForms usage. Extending this later is still possible but will be driven by necessity.
xf-instance The instance contains the data of a form which can be updated during runtime. one or many
xf-bind Defines a binding to one or more instance data nodes. Can define constraints and calculations for the bound nodes one or many
xf-submission Defines a submission that selects, validates and sends instance data to defined endpoints. one or many
